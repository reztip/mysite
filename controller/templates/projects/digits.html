Digit Recognizer in Jupyter Notebook¶

In this Kaggle project, I use supervised learning algorithms to classify black and white handwritten digits as integers from 0 - 9. I am using Python's scikit-learn library for training classifiers.

This is a classic problem in Machine Learning that has provided tremendous cost savings to the shipping industry. Kaggle has the following to day about the dataset:

The data for this competition were taken from the MNIST dataset. The MNIST ("Modified National Institute of Standards and Technology") dataset is a classic within the Machine Learning community that has been extensively studied. More detail about the dataset, including Machine Learning algorithms that have been tried on it and their levels of success, can be found at http://yann.lecun.com/exdb/mnist/index.html.

This is a great first problem to get started with statistical machine learning and predictive modeling. The dataset is fairly robust and easy to understand. It is difficult to think of additional features useful for this model, so the exercise allows you to focus on the performance of different types of classifiers.

I approach the problem with the following classifiers:

  • Source Vector Machine
  • Random Forest Classifier
  • K-Neighbors Classifier and K Means
  • AdaBoost

All of these are great for supervised learning, but we might expect the second and third classifiers to perform better than the first. SVM is typically used to divide N-dimensional planes up into categories (corresponding to digits). K-Neighbors classifier groups items by feature similarity, and RandomForestClassifier is an ensemble method that takes the most-likely output from a forest of decision trees. I would intutively expect a K-Neighbors or K-Means classifer to perform well, since it groups data based on how similarity.

Since the dataset is robust, all of these are likely to perform decently well. This most important step of this analysis is to make sure that our models are cross-validated and our classifier parameters are optimized, so that the training results extend well to the test set.

The Dataset¶

Kaggle has the following to say about the dataset:

Each image is 28 pixels in height and 28 pixels in width, for a total of 784 pixels in total. Each pixel has a single pixel-value associated with it, indicating the lightness or darkness of that pixel, with higher numbers meaning darker. This pixel-value is an integer between 0 and 255, inclusive.

Kaggle offers a training dataset (for fitting a classifier) and a testing dataset, on which you make predictions and submit the results to Kaggle, which scores your output using the data labels they have on file.

Each row of the file train.csv is 785 columns: one label column (the actual correct digit classification, followed by 784 pixel columns (pixelx for $x \, \epsilon \, \{1,..,783\}$). Each of these pixel features contains the pixel intensity of that image, which is a number 0 to 255, inclusive. This indicates the relative blackness found in the original image.

In order to use traditional image plotting functions in Python, these rows must be converted into 28 x 28 numpy arrays.

Classification¶

First try a naive classification with a RandomForestClassifier, and see how it performs on the training and test sets. A RFC is an ensemble classifier that takes in many decision trees, and picks the most likely classification value from the totality of decision trees. You can read about decision trees here.

We will see how our classifier performs using scikit-learn's accuracy_score function, which is how the digit recognition problem is scored. For the record, it takes in a a list of true labels and a list of predictions and returns the percentage classified correctly.

Let us do some parameter optimization using SearchCV algorithms to see if we can do better with a RandomForestClassifier. By default it uses 3-fold cross validation, which basically cycles test and train dataset until it finds a model that works best. It splits the data into three pieces, and trains the data on two of the pieces, testing on the other. This is then repeated so that all three pieces are tested. The searcher then selects the classifier with parameters that give the best accuracy score.

It turns out the GridSearch offers no performance advantage over the standard classification, because it does so well on the training set.

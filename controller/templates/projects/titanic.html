Titanic - Kaggle Competition

This is an introduction to data science using an IPython notebook. Here, I explore Kaggle's titanic dataset to create predictions for passenger survival based on a variety of data features. Approximately 2,000 people set sail on the RMS Titanic's maiden voyage in April of 1912. On April 12, the ship hit an iceberg in the North Atlantic ocean. Here is what Wikipedia has to say on the matter:

Of the 2,224 passengers and crew aboard, more than 1,500 died in the sinking, making it one of the deadliest commercial peacetime maritime disasters in modern history.

The Competition

Kaggle is hosting this introductory competition to get people interested and involved in data science. In this analysis, you will use a training dataset to build a model that can predict passenger survival based on various attributes. Then you will apply the model to the testing dataset to predict the survival of the 417 passengers in that file. Kaggle will then evaluate your predictions using the actual survival data of the test passengers.

The Dataset


Our training dataset will have data for 891 passengers. The dataset has the following features:

  • PassengerId: The identifier of the passenger on the ship
  • Survived: Whether or not the passenger survived (1 = survival)
  • Pclass: The passenger's class (1st, 2nd or 3rd class).
  • Name: The passenger's name, obviously.
  • Sex: The gender of the passenger (male or female)
  • Age: The age of the passenger in years.
  • SibSp: How many siblings or spouses the passenger on board.
  • Parch: The number of parents and children the passenger on board.
  • Ticket: The ticket number for the passenger. I do not use this field.
  • Fare: The fare paid by the passenger for the trip.
  • Cabin: The cabin number of the passenger's room. I do not use this field.
  • Embarked: The embarkation point for the passenger ( C = Cherbourgh, Q = Queenstown, S = Southampton)

The test dataset has all the same features, except for whether or not the passenger survived.


The Process


In a data analysis project, there are several major steps you have to take:

  1. Understanding the structure of your dataset
  2. Cleaning the dataset: missing values, outliers, breaking complex data into pieces
  3. Creating new data features out of the data
  4. Exploratory data analysis and visualization
  5. Rinse and repeat the above steps until satisfied
  6. Build a predictive model
  7. Test your predictive model
  8. Optimize and choose your best model: Deciding what classifier to use, what features are important or helpful, how to optimize your classifier parameters
  9. Rinse and repeat as satisfied/until you have to move onto something else

This is an intentionally vague description, because it is a process you develop an intuition for after extended practice. This is, after all, an introductory analysis, but should give you a clue about the major cycles that occur during the analysis process.

Feature Engineering/Creation

Ok, now let's create some features using our existing dataset. First order on the list is to convert text variables into numeric variables, so that our predictive models will be able to use the features in their algorithms.

Highlights:

  • I created a Gender variable, which maps male to 0 and female to 1
  • I created three dummy variables Embarked_from_x for x in {C,Q,S}
  • Create dummy variable for each class: FirstClass, SecondClass, ThirdClass
  • Create variable Family for size of family: SibSp + Parch
  • Create multiple binary variables for gender x class x embarkation point combination
  • I create a Child variable to indicate if someone is less than 13 years old (L'Chaim?) - graph above indicates that childhood is a good indicator of survival - women and children first!

Truncating the dataset

This outlines the general approach for how use the sklearn library.

  1. Translate our dataset to numpy arrays (as above)
  2. Select some K features out of our total number to use in the process
  3. Transform the datasets (test and train) to only use the K best features
  4. Choose a type of classifier (I use a RandomizedForestClassifier
  5. Optimize the parameters from the classifier using a gridsearch algorithm (RandomizedSearchCV
  6. Fit the optimized classifier to the training dataset
  7. Generate predictions from the test dataset
  8. Output the predictions to a file (named survival_predictions.csv here)

First, select some number K (25) best features from our dataset. Create a testing dataset using these number of features.
